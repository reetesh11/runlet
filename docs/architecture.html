Run Lifecycle (T1–T9)

T1
Fetch Config
Load deployment, decrypt config with CONFIG_ENCRYPTION_KEY, bind connectors
T2
Input Guardrails
Check PII policy, topic blocklist, content filters. Block or mask before LLM sees it.
T3
LLM Call
Call Claude with system prompt + input. Track tokens, latency, cost.
T4
Output Guardrails
Score confidence. Check output content. Block if guardrails fire.
T5
Connector Actions
Execute Zendesk/Slack/GitHub/Notion actions from structured LLM output.
T6
Human Review Gate
If confidence < threshold, pause run. Queue human_review_request. Wait for decision.
T7
Audit Write
Write all events to audit_events table. Immutable trail of every action.
T8
Payload Storage
Encrypt + store input/output JSON to Cloudflare R2. Store reference in DB.
T9
Completion
Update run status, duration, token count. Increment deployment.runCount. Send alerts if configured.

Key Design Decisions

Agents are marketplace-level
No workspace_id on agents table

Agents are public catalogue items. The workspace_agents table links a workspace to an installed agent. Mirrors npm — packages are global, your package.json is local.

JWT over database sessions
Sessions never written to DB

Zero DB load from session reads. Works if Postgres is slow. Trade-off: can't force-logout users server-side. Switch to database strategy when you need that.

Proxy architecture for API auth
Browser never sends tokens to Hono

All API calls go through /api/v1/ in Next.js. The proxy decrypts JWT and adds X-Internal-Secret. Hono stays stateless. In production Hono can be locked to Vercel IPs only.

postgres.js over node-postgres
Recommended by drizzle-orm docs

The pg package has compatibility issues with newer drizzle-orm. postgres.js with prepare: false works reliably for both migrations and queries.
